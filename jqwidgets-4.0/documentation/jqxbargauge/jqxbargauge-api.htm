Properties

Name Type Default
columns Array null

Sets or gets the kanban columns.

Code examples 


$("#jqxKanban").jqxKanban(
{
    source: source,
    columns: [
                    { text: "Backlog", dataField: "new", restrictions: "none", maxItems: 5 },
                    { text: "In Progress", dataField: "work", restrictions: "none", maxItems: 5 },
                    { text: "Testing", dataField: "test", restrictions: "none", maxItems: 5 },
                    { text: "Done", dataField: "done", restrictions: "none", maxItems: 5 }
    ]
});

Column Properties:
  • text - sets the text in column's header.
  • datafield - sets the column datafield.
  • restrictions - sets restrictions to column. // not yet included
    • 'none' - default value. There is no restrictions about this column.
    • 'dragOnly' - user can only drag cards from this column.
    • 'dropOnly' - user can only drop cards in this column.
    • 'restricted' - this column can't be used for drag/drop cards.
  • maxItems - sets maximum number of items per column. // not yet included
Try it: columns is set to a custom array

source Array null

The source object represents a set of key/value pairs. Every set is kanban item.

Code examples

Set the source property. 


$("#jqxKanban").jqxKanban(
{
    columns: columns,
    source: [
                    { id: 0, status: "new", text: "Cookies", content: "Content", tags: "cookies", color: "lightgreen", personId: 1, $css: "standard" },
                    { id: 1, status: "done", text: "Kitchen adventurer caramel", tags: "kitchen, adv, caramel", color: "lightgreen", content: "content", personId: 2, $css: "standard" }
    ]
});

Source item Properties:
  • id - unique identifier of the item. This property is required!
  • status - sets the column where the item will be stored. Associated to column's dataField property. This property is required!
  • text - sets the text area. Could be used as text or headline.
  • content - sets the content area. Could be filled with text/images/widgets etc.
  • tags - sets taggs stored in item footer div.
  • color - sets unique status color of the item.
  • personId - contains person id, associated to some person from staff.
  • $css - sets individual css class about item.

Get the source property. 

var source = $('#jqxKanban').jqxKanban('source');
Try it: source is set

theme String ''

Sets the widget's theme.

jQWidgets uses a pair of css files - jqx.base.css and jqx.[theme name].css. The base stylesheet creates the styles related to the widget's layout like margin, padding, border-width, position. The second css file applies the widget's colors and backgrounds. The jqx.base.css should be included before the second CSS file. In order to set a theme, you need to do the following:
  • Include the theme's CSS file after jqx.base.css.
    The following code example adds the 'energyblue' theme. 
    

    <link rel="stylesheet" href="../../jqwidgets/styles/jqx.base.css" type="text/css" />
<link rel="stylesheet" href="../../jqwidgets/styles/jqx.energyblue.css" type="text/css" />
  • Set the widget's theme property to 'energyblue' when you initialize it.
Try it: theme is set to 'energyblue'
